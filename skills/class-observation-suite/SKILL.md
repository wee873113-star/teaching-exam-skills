# 觀議課全套表單自動化技能 (class-observation-suite)

## 技能概覽

本技能為觀議課相關文書作業的終極整合方案。藉由提供「活動設計單」、「觀課紀錄表」與「6 張照片」，本技能能一次完成並產出全套 **5 份公開授課必備檔案**，並統一將新填入資料儲存於「完成檔」資料夾中，格式化字型為「微軟正黑體」、12號字，且保證照片集檔案剛好呈現在單一頁面內。

### 整合生成的 5 份檔案與邏輯：

1.  **【115-1】觀察前會談紀錄表**
    *   自動填寫授課教師、年級、科目、單元、觀課人員。
    *   自動計算前會談日期為「觀課日期 - 7 天」。
    *   自動嵌入活動設計單中的完整準備、發展、綜合活動文字。
    *   更新配合事宜中的觀課日期、室外場地與回饋會談日期。
2.  **【115-1】觀察紀錄表**
    *   自動填寫教師、年級、科目、單元、觀課人員、觀課日期與室外地點。
3.  **【115-1】觀察後回饋會談紀錄表**
    *   自動填寫基本中繼資料。
    *   自動計算回饋日期為「觀課日期 + 7 天」。
    *   依據觀課紀錄，自動彙整教師四大教學優點、專業收穫與下次教學行動策略（含下次觀察焦點）。
4.  **【115-1】授課教師自評表**
    *   自動填寫基本欄位與觀課日期。
    *   智慧合成「實際教學活動簡述」與「學生課堂表現」。
    *   條列學習目標達成情形、自我省思與同儕回饋後心得。
5.  **【115-1】教師同儕學習活動照片**
    *   依序從 01 至 06 填入 6 張照片，將照片主檔名作為說明文字。
    *   套用一頁式極限優化（單元格前後間距歸零、尾隨空白段落縮至 1pt 且字型 1pt）。
    *   照片寬度預設設定為極限值 **`3.3` 吋**（高度等比例縮小為 `2.48` 吋），確保在不溢出至第二頁的前提下，照片面積最大化。

## 處理流程

### Step 1：準備來源檔案

將所有來源 Word 檔與照片放置在同一個資料夾中（例如：`C:\antigravity\115觀議課`）：
*   來源文檔：`活動設計單.docx`、`觀課紀錄.docx`
*   模板文檔：`觀察前會談紀錄表.docx`、`觀察紀錄表.docx`、`觀察後回饋會談紀錄表.docx`、`授課教師自評表.docx`、`教師同儕學習活動照片.docx`
*   照片資料夾：命名為 `照片`，內含 6 張檔名以 01~06 開頭的 JPG/PNG 圖片。

### Step 2：執行整合腳本

呼叫位於本技能目錄下的 Python 腳本 `complete_suite.py`：
```bash
# 生成全套 5 份檔案
python C:\antigravity\0601math\skills\class-observation-suite\scripts\complete_suite.py --dir "資料夾路徑"

# 僅填寫 4 份文檔表單 (不包含照片)
python C:\antigravity\0601math\skills\class-observation-suite\scripts\complete_suite.py --dir "資料夾路徑" --only-forms

# 僅填寫 1 份照片文檔 (不包含表單)
python C:\antigravity\0601math\skills\class-observation-suite\scripts\complete_suite.py --dir "資料夾路徑" --only-photos
```

## 腳本說明

整合腳本主程式位於：[complete_suite.py](file:///C:/antigravity/0601math/skills/class-observation-suite/scripts/complete_suite.py)

可選參數：
- `--dir`：觀議課檔案資料夾路徑（必填）
- `--font`：文字欄位填入字型（預設：微軟正黑體）
- `--size`：文字欄位字型大小（預設：12）
- `--photo-width`：照片集照片寬度（預設：3.3 吋）
- `--search-width`：若加上此參數，會啟動 Word COM 引擎動態搜尋最佳一頁式寬度。
