# 國小數學互動小遊戲產生器 (teaching-minigames)

將教材自動轉換為「一重點一小遊戲」的國小形成性評量網頁，附 QR Code 讓教師在任何數位平台（如 Line、Padlet、Google Classroom）快速分享與投影使用。

---

## 工作流程

```
1. 分析國小教材 → 2. 提取重點清單（確認） → 3. 製作互動小遊戲 HTML
→ 4. 打包索引頁 → 5. 自動上傳 GitHub → 6. 啟用 GitHub Pages
→ 7. 回傳每個遊戲的網址 + QR Code
```

## 🎯 國小版小遊戲設計規範

為了符合國小學生的閱讀與心理特徵，製作遊戲 HTML 時需嚴格遵守以下視覺與互動規範：

### 1. 視覺設計 (Kid-Friendly UI)
*   **字體放大**：小學生閱讀字體需偏大，內文字型大小建議為 `18px` 到 `22px`，標題為 `24px` 以上。
*   **按鈕大而好按**：觸控按鈕高度至少為 `50px`，間距加寬，方便學生在手機或平板上點擊。
*   **明亮活潑色彩**：背景使用粉粉嫩嫩的色彩（如軟萌黃 `#fff9db`、粉藍 `#e7f5ff`、粉綠 `#ebfbee`），卡片保持白色，營造溫暖有趣的學習氣氛。

### 2. 回饋與激勵機制 (Positive Feedback)
*   **正向鼓勵語**：答對時，使用熱情的文字與 Emoji（如：「🎉 太棒了！你答對了！✨」、「⭐ 答對了！你超強的！👍」）。
*   **溫和提示語**：答錯時，避免打擊學生信心，使用鼓勵性的字眼（如：「💪 差一點點，再接再厲！正確答案是：...」）。
*   **進度條/星星**：以簡單直觀的圓點、星星或進度條來顯示目前作答進度（如「⭐ 第 2 / 5 題」）。

## 遊戲類型選擇

根據國小單元的性質挑選最合適的遊戲類型：

| 遊戲類型 | 適用國小情境 | 建議檔名 |
|---------|---------|------|
| 選擇題 (MCQ) | 概念判定、四則乘除運算選擇 | `game-mcq.html` |
| 填充題 (Fill-in-Blank) | 單元名詞填空、公式填空 | `game-fill.html` |
| 配對題 (Matching Pairs) | 連連看（如：算式配對答案、圖形配對名稱） | `game-match.html` |
| 是非題 (True/False) | 迷思概念澄清（如：正方形也是長方形的一種嗎？） | `game-tf.html` |
| 排序題 (Ordering) | 數字大小排列、長度/容量由小到大排列 | `game-order.html` |

---

## 注意事項

- **無外部依賴**：考量學校電腦教室或平板網路可能受限，所有小遊戲的 CSS、JS 必須**全部內嵌於單一 HTML 檔案中**，不使用任何外部 JS/CSS CDN 連結（如 jQuery 或 Bootstrap），確保 100% 離線可用。
- **純淨無廣告**：動態生成的 QR Code 使用免費無廣告 API。
- **文字淺顯**：遊戲介面上的說明文字（如「請點擊正確答案開始」、「請輸入你的答案」）應盡量口語且簡短。
